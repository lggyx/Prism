## 0. 当前产品事实与接口范围

### 0.1 已确认核心流程

```text
登录/注册
  ↓
收藏页 / 空状态
  ↓ 点击 CAPTURE
A+B 取景到镜片选择
  ↓ 拍照上传
选择镜片
  ↓ 创建 AI 重述任务
C 重述结果页
  ↓ 保存切片
D 收藏页
```

### 0.2 当前前端页面覆盖

| 页面 | 文件 | 后端依赖 |
|---|---|---|
| Splash 启动页 | `0-1 splash.html` | 客户端本地状态；可选 `/client-config` |
| Auth 登录注册 | `H auth.html` | 验证码、登录/注册、协议 |
| Onboarding 新用户引导 | `0-3 onboarding.html` | 登录接口返回 `nextRoute=onboarding` |
| A+B 取景到镜片选择 | `B+A capture-to-lens.html` | 拍照上传、镜片列表、AI 解析任务 |
| C 重述结果 | `C lens-result.html` | 重述结果详情、保存切片 |
| D 收藏页 | `D collection-negative.html` | 切片列表、详情、删除、重新解析、导出 |
| J 收藏空状态 | `J collection-empty.html` | 切片列表返回空、统计返回 0 |
| X-2 收藏筛选空状态 | `X-2 collection-filter-empty.html` | 切片筛选列表返回空 |
| E 镜片库 | `E lens-library.html` | 镜片列表、详情、示例切片、自定义镜片 |
| B-1 语聊创建镜片 | `B-1 voice-lens-creator.html` | `/lens-creator/*` |
| F 发现页 | `F discover.html` | 发现聚合、挑战、热门镜片、社区流、共鸣、举报 |
| X-5 发现空状态 | `X-5 discover-empty.html` | 社区流返回空 |
| G 个人页 | `G profile.html` | 用户统计、配置、登出 |
| X-1 AI 兜底状态 | `X-1 ai-reading-fallback.html` | `Reading.status=failed/empty/timeout` |
| X-3 权限解释页 | `X-3 permission-request.html` | 可选 `/client-config` |
| X-4 分享预览页 | `X-4 share-preview.html` | 导出任务 |

### 0.3 UX 报告要求后端支持的边界状态

| 状态 | 后端支持方式 |
|---|---|
| AI 解析失败/超时 | `Reading.status = failed/timeout` + `failureReason` |
| AI 内容为空 | `Reading.status = empty` + `emptyReason` |
| 收藏 0 切片 | `GET /slices` 返回 `total = 0` + `emptyState` |
| 筛选后无结果 | `GET /slices?lensId=...` 返回 `total = 0` + `emptyState` |
| 社区坐标模糊 | 社区接口只返回 `locationText`，其值由 `GeoLocation.publicText` 派生 |
| 用户关闭社区位置 | 社区接口返回 `locationText = null`，不返回经纬度 |
| 图片上传/保存 loading | 接口幂等，返回可轮询任务状态 |
| 自定义镜片语音创建 | 语音上传、转写、对话轮次、镜片草稿确认 |

## 1. 全局约定

### 1.1 Base URL

```text
生产: https://api.worldview-lens.example.com/api/v1
开发: http://localhost:3000/api/v1
```

### 1.2 鉴权

```http
Authorization: Bearer <accessToken>
X-Device-Id: <deviceUuid>
X-Client-Version: 0.1.0
```

说明：

1. 登录前允许匿名设备态，服务端可基于 `X-Device-Id` 分配临时 `observerCode`。
2. 登录后返回正式 `accessToken` / `refreshToken`。
3. MVP 社区身份使用 `OBS-XXXX`，不展示头像和昵称。

### 1.3 通用响应

成功：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

失败：

```json
{
  "code": "PRISM_VALIDATION_ERROR",
  "message": "email is invalid",
  "requestId": "req_20260517_xxx"
}
```

### 1.4 分页

请求：

```text
limit: 默认 20，最大 50
cursor: 上一页返回的 nextCursor
```

响应：

```json
{
  "items": [],
  "nextCursor": "cur_xxx",
  "hasMore": true
}
```

### 1.5 时间与坐标

| 类型 | 约定 |
|---|---|
| 时间 | ISO 8601，带时区，例如 `2026-05-17T01:30:00+08:00` |
| 私人坐标 | 收藏页/个人数据可返回精确坐标 |
| 社区坐标 | 只返回城市级或一位小数模糊坐标 |
| 前端标注坐标 | 使用 0-1 相对坐标，不使用像素坐标 |

### 1.6 命名规范

| 类型 | 规范 |
|---|---|
| 资源主键 | 详情对象内统一使用 `id`，例如 Reading 对象返回 `id: "read_01"` |
| 创建任务响应 | 可同时返回语义化 id 字段，例如 `sliceId`、`exportTaskId`，方便前端直接取用 |
| 私人位置 | 使用 `location.privateText`，仅用户自己的收藏/详情可见 |
| 社区位置 | 使用扁平字段 `locationText`，值来自 `GeoLocation.publicText`；关闭展示时为 `null` |
| 路由提示 | `nextRoute` 只使用 `onboarding`、`collection`、`capture`、`lens-picker`、`lens-result` |
| 空状态 | 列表接口统一在 `data.emptyState` 返回空状态文案；非空时为 `null` |

## 2. 并行开发模块拆分

| 模块 | Owner 建议 | 可独立开发接口 |
|---|---|---|
| A. 认证与用户 | 后端 1 | `/auth/*`、`/me`、`/me/settings` |

```text
Auth → 所有模块
```

## 4. A 模块：认证与用户

当前 `H auth.html` 使用邮箱 + 验证码；项目进度总结中曾提到手机号验证码。为避免返工，接口统一设计为 `channel + target`，同时支持邮箱和手机号。

### 4.1 发送验证码

```http
POST /auth/code
Content-Type: application/json
```

请求：

```json
{
  "channel": "email",
  "target": "alex@example.com",
  "scene": "login"
}
```

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| channel | `email` / `sms` | 是 | 当前原型用 `email` |
| target | string | 是 | 邮箱或手机号 |
| scene | `login` / `bind` | 否 | 默认 `login` |

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "cooldownSeconds": 60,
    "expiresInSeconds": 300,
    "maskedTarget": "a***@example.com"
  }
}
```

3. MVP 只支持 `interfaceTheme = DARK`。

## 5. B 模块：拍摄资产与 AI 重述

### 5.1 上传拍摄照片

`B+A capture-to-lens.html` 中快门后进入镜片选择。真实 App 中应先本地完成拍摄，再上传图片获得 `captureId`。

**约束**：图片最大 10 MB，支持 `image/jpeg`、`image/png`、`image/webp`、`image/heic`。超限返回 `PRISM_IMAGE_TOO_LARGE`。

```http
POST /captures
Content-Type: multipart/form-data
```

请求字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| image | File | 是 | 拍摄照片 |
| capturedAt | string | 是 | 拍摄时间 |
| latitude | number | 否 | 纬度 |
| longitude | number | 否 | 经度 |
| accuracyMeters | number | 否 | 定位精度 |
| iso | number | 否 | 仪器参数 |
| shutter | string | 否 | 如 `1/60s` |
| aperture | string | 否 | 如 `f/1.8` |

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "cap_01",

    "height": 4032,
```

## 10. 错误码

| 错误码 | HTTP | 说明 |
|---|---:|---|
| `PRISM_UNAUTHORIZED` | 401 | 未登录或 token 失效 |
| `PRISM_FORBIDDEN` | 403 | 无权访问资源 |
| `PRISM_NOT_FOUND` | 404 | 资源不存在 |
| `PRISM_VALIDATION_ERROR` | 400 | 参数校验失败 |
| `PRISM_CODE_TOO_FREQUENT` | 429 | 验证码发送过于频繁 |
| `PRISM_CODE_INVALID` | 400 | 验证码错误或过期 |
| `PRISM_IMAGE_TOO_LARGE` | 413 | 图片过大 |
| `PRISM_UNSUPPORTED_IMAGE` | 415 | 不支持的图片格式 |
| `PRISM_READING_FAILED` | 500 | AI 解析失败 |
| `PRISM_READING_TIMEOUT` | 504 | AI 解析超时 |
| `PRISM_DUPLICATE_SAVE` | 409 | 重复保存切片 |
| `PRISM_RATE_LIMITED` | 429 | 请求过于频繁 |

说明：AI 无有效标注不是错误响应，统一使用 `GET /readings/{readingId}` 的业务状态 `status = "empty"`。

## 13. AI 解析生成约束

后端调用多模态模型时应遵守：

1. 不强制判断“主体”，只标注画面中存在的项目。
2. 默认返回 5 条标注，MVP 可接受 3-5 条。
3. 标注文字建议 16-36 个中文字符，最多两行。
4. 坐标使用 0-1 相对坐标。
5. 总结语必须体现镜片世界观，不写成通用图片描述。
6. 社区接口永远不返回精确坐标。
7. 对低质量图片返回 `status=empty`，不要硬编标注。
