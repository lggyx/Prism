## 二、缺失页面（必须新增）

### 2.1 启动页 / 闪屏（Splash）

**优先级：高**

当前缺失。用户首次打开 App 的第一屏。

需要内容：
- App Logo + 名称"世界观透镜"
- Slogan："换一副镜片，重新看见世界"
- 加载动画（可用透镜光圈开合的动效，和 iris close 呼应）
- 1-2 秒后自动跳转到登录页或主页

### 2.2 登录 / 注册页（Auth）

**优先级：高**

当前完全缺失。用户进入 App 后的身份认证环节。

需要内容：
- 手机号 + 验证码登录（国内主流方案）
- 输入框：手机号、验证码
- "获取验证码"按钮（60 秒倒计时）
- "登录 / 注册"按钮（新用户自动注册）
- 底部：用户协议 + 隐私政策链接
- 视觉风格：延续深色仪器面板风格，输入框用细边框等宽字体

可选（MVP 可不做）：
- 第三方登录（微信）
- 跳过登录（游客模式）

### 2.3 新用户引导页（Onboarding）

**优先级：中**

首次注册后的 2-3 屏引导，帮助用户理解产品核心概念。

建议内容（可左右滑动的卡片）：
- 第 1 屏："拍下你看到的" — 取景页的示意
- 第 2 屏："选一副镜片" — 镜片选择的示意
- 第 3 屏："世界被重新讲述" — 标注结果的示意
- 底部："开始观察"按钮，跳转到主页

### 2.4 AI 语聊创建镜片面板（Voice Lens Creator）

**优先级：中**

从镜片选择页底部"+ 创建自定义镜片"入口触发，从底部弹起的面板或全屏页。

需要内容：
- 顶部提示语："描述你想要的观察视角"
- 中央：大号语音按钮（按住说话 或 点击切换录音状态）
- 对话区域：显示 AI 引导问题 + 用户语音转文字结果（气泡式）
- AI 引导流程（3-5 轮）：
  1. AI："你想用什么样的视角看世界？"
  2. 用户回答
  3. AI 追问细化维度
  4. AI 生成镜片定义（名称 + 描述 + 关注维度），展示预览卡片
  5. 用户确认 → 镜片创建完成，直接可用
- 底部："完成创建"按钮 + "取消"
- 录音状态指示：波形动画或脉冲圆环

### 2.5 切片分享页 / 导出预览（Share Preview）

**优先级：低**

用户在收藏页或重述结果页点击"分享"后的预览。

需要内容：
- 将认知切片渲染为一张可分享的图片（照片 + 标注 + 总结语 + 镜片标识 + 坐标日期）
- 底部分享渠道：保存到相册 / 微信 / 微博等
- 底部附带 App 水印或二维码

## 三、现有页面的 UX 问题与缺失状态

### 3.1 A 取景页（shooting.html）

| 问题 | 说明 | 优先级 |
|------|------|--------|
| ⚠️ GPS 坐标未动态显示 | 设计规范要求左下角显示实时 GPS 坐标 + 日期，当前页面只有仪器参数 ISO/快门，缺少坐标和日期 | 中 |
| ⚠️ 闪光灯按钮缺失 | 设计规范要求右上角有闪光灯切换，当前无此控件 | 低 |
| ⚠️ 关闭按钮缺失 | 设计规范要求左上角有关闭/返回按钮，当前只有状态指示 | 中 |
| ✅ iris close 动效 | 实现质量高，和设计规范一致 | — |
| ⚠️ 拍照后底部出现镜片卡片 | 当前实现是拍照后底部直接出现镜片横向卡片列表，和 B 镜片选择页功能重叠。需要确认：是 A 页面拍完直接选镜片，还是跳转到 B 页面选 | 高 |

**关键决策点**：A 页面的底部镜片卡片列表和 B 页面的镜片选择是两套独立实现。在实际 App 中，A 拍完应该通过 iris close 过渡到 B 页面，而不是在 A 页面内选镜片。当前 A 页面的镜片卡片可以视为"快速选镜片"的 shortcut，但这和已确认的"两步分离"设计决策冲突。**建议：A 页面去掉底部镜片卡片，iris close 后直接跳转 B 页面。**

## 四、页面间跳转逻辑（Flow Map）

### 主流程

```
闪屏 → 登录/注册 → [首次] 新用户引导 → 主页（收藏页）
                                          ↓
                              Tab Bar 切换各页面
```

### 核心流程跳转

```
收藏页/任意 Tab ──点击 CAPTURE Tab──▶ A 取景页
A 取景页 ──快门 + iris close──▶ B 镜片选择页
B 镜片选择页 ──点击镜片──▶ C 重述结果页
C 重述结果页 ──保存切片──▶ 数据写入收藏页
C 重述结果页 ──换镜片──▶ B 镜片选择页（保留同一张照片）
C 重述结果页 ──返回──▶ B 镜片选择页
B 镜片选择页 ──返回──▶ A 取景页（或直接回主页？需确认）
B 镜片选择页 ──创建自定义──▶ AI 语聊面板（2.4）
```

### Tab 间跳转

```
COLLECT（收藏页）⟷ LENSES（镜片库）⟷ CAPTURE（取景页）⟷ DISCOVER（发现页）⟷ PROFILE（个人页）
```

### 跨页面入口

```
E 镜片库详情 ──"USE THIS LENS"──▶ A 取景页（预选该镜片）
F 发现页切片 ──"USE SAME LENS"──▶ A 取景页（预选该镜片）
F 发现页挑战 ──"JOIN"──▶ A 取景页（预选挑战指定镜片）
D 收藏页切片 ──"REANALYZE"──▶ B 镜片选择页（带着原照片）
```

## 五、缺失的全局状态与边界情况

### 5.1 网络与加载状态

| 状态 | 出现场景 | 当前 | 需要 |
|------|----------|------|------|
| AI 解析加载中 | B→C 过渡时 | ✅ 有扫描线 | ✅ |
| AI 解析失败 | B→C，模型超时或出错 | ❌ 缺失 | 需要：错误提示 + "重试"按钮 |
| AI 解析内容为空 | 照片模糊/无可识别内容 | ❌ 缺失 | 需要：友好提示"这片视野太模糊了，试试靠近一点？" + 重拍按钮 |
| 网络断开 | 任意需要联网的操作 | ❌ 缺失 | 需要：全局 toast "网络连接已断开" |
| 社区内容加载中 | F 发现页首次打开 | ❌ 缺失 | 需要：骨架屏或加载指示 |
| 图片上传中 | 保存切片时 | ❌ 缺失 | 需要：保存按钮 loading 状态 |

### 5.2 空状态

| 页面 | 场景 | 当前 | 需要 |
|------|------|------|------|
| D 收藏页 | 0 切片 | ❌ 缺失 | "还没有认知切片" + "开始观察"按钮 |
| D 收藏页 | 筛选后无结果 | ❌ 缺失 | "该镜片下暂无切片" |
| E 镜片库详情 | 某镜片 0 次使用 | ⚠️ 需确认 | "NO READINGS YET" + "CAPTURE WITH THIS LENS" |
| F 发现页 | 社区无内容（极端情况） | ❌ 缺失 | "暂无信号，成为第一个观察者" |

### 5.3 权限请求

| 权限 | 触发时机 | 当前 | 需要 |
|------|----------|------|------|
| 相机权限 | 首次点击 CAPTURE | ❌ 缺失 | 系统权限弹窗前的解释页（为什么需要相机） |
| 麦克风权限 | 首次点击"创建自定义镜片" | ❌ 缺失 | 系统权限弹窗前的解释页（为什么需要麦克风） |
| 定位权限 | 首次拍照 | ❌ 缺失 | 系统权限弹窗前的解释页（为什么需要位置） |
| 相机权限被拒绝 | 用户拒绝了相机权限 | ❌ 缺失 | 引导用户去设置中开启 |
