## 一、项目定位

**世界观透镜**是一款移动端 App，核心体验是：用户拍下一张现实照片，选择一个“世界观镜片”，AI 用该视角重新解释画面，并以田野笔记式的标注线和文字叠加在照片上。每次解读可以保存为一枚“认知切片”，逐步形成用户自己的视角档案。

一句话定位：

> 不是帮用户识别世界，而是帮用户收藏视角、扩展视角、重新理解世界。

核心链路：

```text
拍下现实
  → 选择镜片
  → AI 世界重述
  → 保存认知切片
  → 沉淀到收藏页
```

## 三、已完成内容

### 3.1 产品与 UX 设计

已明确产品主流程、页面体系、状态覆盖和视觉语言。

视觉方向：

| 维度 | 当前决策 |
|---|---|
| 设计语言 | 高科技光学仪器 |
| 背景 | 深色沉浸 `#0a0c10` |
| 字体 | 元数据、导航、标签使用等宽字体 |
| 圆角 | 4-6px，小圆角、精密感 |
| 信息组织 | 仪器面板、细线分割、底片浏览器 |
| 社交表达 | 弱化头像昵称，使用 OBS 编号与“◇ 共鸣” |

核心设计决策：

| 主题 | 结论 |
|---|---|
| 图片来源 | 仅实时拍照，不接入相册 |
| 拍照与选镜片 | 同一页面内连贯过渡，使用 `B+A capture-to-lens.html` |
| 镜片选择 | MVP 单选，预设镜片优先 |
| AI 解读 | 不强调主体识别，而是标注画面中可观察项目 |
| 标注呈现 | 圆点、线、文字逐条出现 |
| 保存动效 | 闪白，模拟截屏感 |
| 收藏页 | 统一 3:4 底片网格，避免社媒瀑布流 |
| 位置隐私 | 私人精确坐标，社区模糊到城市级 |

### 3.2 页面完整性

页面级 UX 已补齐。当前 HTML 原型覆盖：

| 模块 | 页面 |
|---|---|

| 核心体验 | 取景、镜片选择、AI 重述结果 |

| 权限 | 权限解释页 |

